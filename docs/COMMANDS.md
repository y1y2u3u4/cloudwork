# Telegram Commands Reference

CloudWork Telegram Bot 命令参考文档。

## 基础命令

### `/start`

显示帮助信息和可用命令列表。

```
/start
```

### `/run <prompt>`

独立执行一个任务，不影响当前会话上下文。

```
/run 帮我创建一个 Python Hello World 程序
/run 分析 ./src 目录的代码结构
```

**特点:**
- 每次 `/run` 是独立任务，不保留上下文
- 适合一次性查询或独立任务

## 会话管理

### `/sessions`

查看当前活跃会话列表，支持点击切换。

```
/sessions
```

**显示内容:**
- 会话名称 (AI 自动命名)
- 最后活跃时间
- 消息数量

### `/new [名称]`

创建新会话。

```
/new                    # 自动命名
/new 前端开发项目        # 自定义名称
```

**特点:**
- 新会话自动成为当前活跃会话
- 如不指定名称，首条消息后 AI 自动命名

### `/archived`

查看已归档的会话，支持恢复。

```
/archived
```

**归档规则:**
- 30 分钟无活动自动归档
- 点击可恢复为活跃会话

### `/delete <会话ID>`

删除指定会话（不可恢复）。

```
/delete abc123
```

## 设置命令

### `/settings`

打开统一设置菜单，包含：
- 模型选择
- 执行模式
- 项目切换

```
/settings
```

### `/model`

切换 Claude 模型。

```
/model
```

**可选模型:**
| 模型 | 说明 |
|------|------|
| `sonnet` | Claude Sonnet - 平衡（默认）|
| `opus` | Claude Opus - 最强大 |
| `haiku` | Claude Haiku - 最快 |

### `/mode`

切换执行模式。

```
/mode
```

**可选模式:**
| 模式 | 说明 |
|------|------|
| `auto` | 自动执行，跳过所有确认（默认）|
| `plan` | 计划模式，先生成计划后执行 |

### `/project`

查看和切换工作空间项目。

```
/project
```

**项目发现规则:**
- 扫描 `workspace/` 目录
- 识别包含 `CLAUDE.md` 的子目录

## 对话方式

### 直接发送消息

在当前活跃会话中继续对话。

```
用户: 帮我写一个排序函数
Claude: [回复...]

用户: 改成快速排序
Claude: [回复，保持上下文...]
```

### 回复历史消息

自动切换到该消息所属的会话。

```
[回复某条历史消息]
用户: 继续上次的工作
```

**特点:**
- 自动切换会话上下文
- 恢复该会话的完整历史

## 交互式问答

当 Claude 需要用户确认时，会显示选择按钮：

```
Claude: 检测到多个文件，请选择要修改的文件：
[文件A] [文件B] [文件C] [取消]
```

点击按钮即可响应。

## 快捷键

| 操作 | 快捷方式 |
|------|---------|
| 取消当前任务 | 发送 `取消` 或 `/cancel` |
| 查看帮助 | `/start` 或 `/help` |

## 示例场景

### 场景 1: 开始新项目

```
/new 电商后端开发
```
→ 创建专用会话
```
帮我设计用户认证模块的数据库结构
```
→ 在该会话中持续开发

### 场景 2: 快速查询

```
/run Python 中如何读取 JSON 文件
```
→ 独立查询，不污染当前会话

### 场景 3: 切换项目

```
/project
```
→ 选择 `frontend-app` 项目
```
帮我修复登录页面的 bug
```
→ Claude 在该项目上下文中工作

### 场景 4: 调试复杂问题

```
/model
```
→ 切换到 `opus` 模型
```
帮我分析这个内存泄漏问题
```
→ 使用更强大的模型处理
